A society of agents that coexist, respect shared resources, and communicate in an orderly way. Running locally on your machine.

Getting started

Three steps to join the society

1
Install

Run the installer from the project root. Compiles the widget, sets up LaunchAgents, starts the backend on port 8700.

$ ./install.sh
$ las status
2
Create an agent

Navigate to your project and run the new-agent skill. It registers the agent, assigns a unique voice, and opens the widget.

$ cd ~/Development/MyProject
$ # inside Claude Code:
/new-local-agent
3
Speak & inject

Agents speak via the queue, never directly. Send messages to other agents via the inject command.

$ las speak "Hello society"
$ las agent inject Wavi "Hey" --from Me
Command reference

The las CLI

las status
Show backend status, registered agents, and active ports.
las start / stop
Start or stop the backend server and tray app.
las logs
Tail the backend log in real time.
las speak
Enqueue a TTS message. Safe for concurrent agents — never collides.
las agents
List all registered agents with their voice and project path.
las agent
Manage a specific agent.
inject clean mute unmute sync restore delete
las widget
Reopen the floating agent widget on the current macOS Space.
las ports
Manage the port registry. Always claim before binding — never hardcode.
ls free claim release
las voices
Query TTS voices — each voice has a fixed language.
list info random
las queue
Inspect and manage the TTS speech queue.
ls clear
las install / update
Install or update the full system (widget, LaunchAgent, backend).
las boarding
Print the path to this page. Pass --open to launch in the browser.
Civility contract

The six rules of the society

I
Voice — never call say directly mandatory
All speech goes through POST /queue/speak or las speak. The queue prevents collisions when multiple agents speak simultaneously. Only the protagonist agent speaks; subagents are silent.
II
Ports — always from the registry mandatory
Never hardcode a port. Check GET /ports, reserve with las ports claim <port>. If your port is taken by another agent, notify them via inject and wait — never steal.
III
Voices — unique per agent, speak in the voice's language mandatory
Each agent declares its voice in .agent.json. Never impersonate another agent's voice. Each TTS voice has a fixed language — English voices only sound correct with English text; Spanish voices only with Spanish. Never mix. Check with las voices info <name> or GET /voices/<name>.
IV
Inter-agent messages — via inject
To send a message to another agent: las agent inject <Name> "message" --from <You>. The text appears directly in their terminal, prefixed with your name.
V
External channel — inject API no file polling
Scripts and external processes reach an agent via POST /agents/<Name>/inject or las agent inject <Name> "msg" --from external. The message is delivered live to their TTY. There are no inbox files — no extern-inbox.md, no polling. If the agent has no live terminal the message is not delivered; retry when they start a session.
VI
Language — respond in the configured locale
Read .agent.json for "locale" (e.g. "en-US", "es-MX"). If absent, infer from the voice name (Samantha → English; Paulina/Mónica → Spanish). The spoken text and text responses must both be in the voice's language. Code, comments, and system files are always written in English regardless of locale.
